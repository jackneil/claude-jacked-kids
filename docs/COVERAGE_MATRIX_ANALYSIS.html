Key observations

The asset — and it's first-of-its-kind. Build+teach for the sweet-spot kid (Make-a-Game 8, Learn 8). Research found no kid-safe Claude Code plugin exists, and no agent on the substrate teaches CS while it builds. The one-liner: raw Claude Code, made warm, safe, teaching, and choreographed for a child.

The real competitor is raw Claude Code. Out of the box it's a software-engineering assistant — dev-toned, no kid-safety, no teaching, blank-prompt, permission-prompt friction; on this machine a global "be blunt / mildly profane" CLAUDE.md would even leak onto the kid (the kid-buddy persona explicitly neutralizes it). Everything this product adds is the missing kid layer; the engine is already there.

Lean on the substrate, don't reinvent it. Claude Code already ships the heaviest items v1 wanted built: /rewind checkpoints + git = the "Manage & Recover" engine (v1's "build auto-checkpoint" was reinventing a built-in); SessionStart hooks = the "new powers" greeting; PreToolUse hooks = a deterministic safety gate; Output Styles = the jargon-strip. The work is kid-skinning these.

The genuine remaining gaps (now that Setup is scored correctly): Play & Show (4.8) + Publish (4.7) — the "show a friend" loop; Identity (4.5) — portfolio/badges; Young Kid (3.7) — pre-reader access (the shipped audio + a future voice-input mode); Teen (5.7) — a fade-to-real-code path so it doesn't feel babyish; Educator (2.8) — a separate product.

Strategic recommendations

Press the wedge: "the only AI that builds your game/app AND teaches you while it does — and keeps it all on your computer." First-of-its-kind on the Claude Code substrate. Lead with it; lean into Make-an-App for young kids (white space even beyond the substrate).

Don't reinvent the substrate. Skin /rewind+git, fire a SessionStart greeting, add a PreToolUse safety gate — free, native, and stronger than bespoke prompt-only versions.

The audience gap is the one real lesson from the standalone platforms. Kids want someone to play their thing — close it with the gated share link (then a moderated opt-in gallery), inside the existing safety model.

Two roles are different products: the Teen (needs fade-to-real-code so it isn't "cheating") and the Educator (seat pricing + rostering + DPA). Each a deliberate, separately-scoped bet.

Plan for portability so the pedagogy isn't Claude-Code-locked: keep skill bodies + persona + README tool-agnostic; isolate the CC-specific glue. Codex CLI is the natural first port.

Profiles

The bar

Raw Claude Code (no plugin)

Anthropic's terminal AI coding agent. It will build a working game/app from a plain prompt — but as a software-engineering assistant: dev tone, no kid-safety, no teaching, no first-win choreography, permission prompts. Output Styles are the documented way to strip the SWE tone; this plugin's persona does that + adds safety + teaching. Output Styles ↗ · Checkpointing/Rewind ↗ · Plugins ↗

Same substrate · port targets

Codex CLI · Gemini CLI · Cursor · Aider

Same engine class, same missing kid layer. Codex CLI shares the open Agent Skills SKILL.md standard (cleanest port). Gemini CLI uses extensions + GEMINI.md. Cursor is a full IDE (more surface for a young kid). Aider is a dev pair-programmer (worst kid-fit). Codex Skills ↗ · Gemini extensions ↗ · Open Agent Skills ↗

The road not taken · audience lesson

Scratch · Code.org · Rosebud AI · Roblox

Standalone kid platforms a family might use instead of the Claude-Code path. They set expectations — web instant-on, remix culture, and an audience/gallery (Scratch: 164M+ projects) — but aren't the per-cell bar for a CC plugin. The audience expectation is the one worth partly meeting. Scratch ↗ · Code.org ↗ · Rosebud AI ↗

Shipped feature · tooling

Audio onboarding (OpenAI TTS)

The "🎧 Listen instead" guides were generated with OpenAI gpt-4o-mini-tts (you own the output; ~1¢; steerable warm tone — OpenAI's terms require disclosing the voice is AI-generated, which the transcript + README links do). Higgsfield's "audio" is a separate voiceover product (reportedly ElevenLabs-backed), so ElevenLabs itself (paid) or OpenAI are the cleaner options; macOS say is the zero-dependency fallback. Generated once, committed as static .mp3 — no runtime TTS, offline-first intact. OpenAI TTS ↗
